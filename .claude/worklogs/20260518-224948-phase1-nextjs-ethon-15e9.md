## 1. research

**Goal:** Stand up a buildable Next.js App Router skeleton carrying the ETHON
design system (tokens, fonts, UI primitives, layout chrome, i18n DE/EN, typed
static data) so Phase 2 can drop in pages.

**Current state:**
- Phase 0 scaffold only; no app code, no Node toolchain.
- Node installed this session: nvm (brew) → Node v22.22.3, pnpm 9.15.4.
  nvm is NOT in non-interactive PATH; every node/pnpm Bash call must first
  `export NVM_DIR="$HOME/.nvm"; \. /opt/homebrew/opt/nvm/nvm.sh`.
- Prototype source reviewed in full (shared/pages/dashboard/auth-admin/
  device-admin/tweaks-panel.jsx + Prototype.html).

**Key findings / decisions:**
- D1: `tweaks-panel.jsx` is a Claude-Design **host-protocol artifact**
  (postMessage `__edit_mode_*`, `/*EDITMODE-BEGIN*/` block, iframe parent). It
  is design-tool scaffolding, not a product feature. The README says recreate
  visual output, not prototype internals that don't fit. → **Drop TweaksPanel
  from the real product.** Language is the real route-based DE/EN header toggle
  (next-intl). `showTicker`/`accent` were design-exploration tweaks → ticker
  shown by default, accent = default blue. (Documented in MEMORY.md.)
- D2: Pin Next 14.2.x + React 18.3.1 (prototype used React 18.3.1; spec said
  "Next.js 14+"). Mature App Router + best next-intl support; avoids React 19
  churn. Exact pins in package.json.
- D3: Scaffold manually (no `create-next-app`) for exact version pins, no
  Tailwind, custom `globals.css`, and the `src/app/[locale]` structure.

**Research summary:**
- Toolchain ready (Node 22 via brew-nvm; must source nvm per Bash call).
- Prototype is authoritative; TweaksPanel intentionally excluded as non-product.
- Manual pinned scaffold, Next 14.2 + React 18.3.
