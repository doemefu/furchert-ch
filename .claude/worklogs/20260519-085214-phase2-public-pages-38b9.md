## 2. plan

**Goal:** Every public route (`/`, `/about`, `/it`, `/rowing`, `/projects`,
`/projects/[slug]`, `/contact`) renders pixel-faithfully to the prototype in DE
and EN with locale-aware metadata; `pnpm lint`/`typecheck`/`build` pass and all
routes static-render (Contact form posts to a real server action).

### Context
**Summary:** Replace the placeholder home and add six public pages by porting
`pages.jsx` layout verbatim (inline styles, `.border-x` grid) into RSC where
possible, with `'use client'` only for the interactivity that remains
(`HoverCard`, Contact form). All prototype body prose is authored **bilingually
(DE+EN)** as locale-keyed typed `src/data/*` modules (user decision A1); short
structural labels go to next-intl. Add `generateMetadata`, `robots.ts` (exclude
`/dashboard`), `sitemap.ts`. Automation is **out of scope** (Phase 3) ‚Äî its nav
link will 404 until then, acceptable per phased plan.

**Assumptions:**

| ID | Assumption | Confidence | Implication if wrong |
|----|------------|------------|----------------------|
| A1 | User wants full EN translations of all body prose | high (explicit) | n/a ‚Äî confirmed via AskUserQuestion |
| A2 | `/projects/[slug]` route replaces in-component selection | high | Would revert to a single client page with state |
| A3 | Real server-action stub (validate+log) satisfies "no fake success" | high | Swap to mailto-only fallback |

**Open questions:** none (Q1 resolved ‚Üí translate everything).

### Options considered

**Option 1 ‚Äî Verbatim layout port, RSC-first, locale-keyed bilingual data (chosen)**
- What: Each page = a server component rendering prototype markup verbatim
  (inline styles kept). All translatable prose authored DE+EN in locale-keyed
  typed `src/data/*` modules (`Record<Locale, T>`); locale-invariant structure
  (tags, icons, slugs, github, status, timeline years) stays plain. Short
  structural labels ‚Üí next-intl. Projects detail ‚Üí real `/projects/[slug]`.
  Contact = client form ‚Üí real server action.
- Pros: Pixel-faithful layout + genuinely bilingual content (CLAUDE.md
  "German + English"); strong typing for structured card/project data;
  RSC-default, SSG-able; reviewable diff (translations co-located per page).
- Cons: Larger diff (EN prose authored); some duplicated inline styles
  (matches prototype; acceptable).

**Option 2 ‚Äî Single client page tree with `setPage` state (rejected)**
- What: Mirror the prototype's SPA hash routing with React state.
- Rejected because: Breaks App Router/SEO/SSG, contradicts CLAUDE.md
  (RSC-default, real routes), no deep links, worse a11y.

**Option 3 ‚Äî All prose in next-intl JSON catalogs (rejected)**
- What: Put bio/cards/projects prose into `de.json`/`en.json`.
- Pros: Single i18n mechanism.
- Rejected because: Card/project content is **structured** (arrays of
  `{title,org,text,items,learning,tags}`); flat JSON + `t.raw()` loses
  TypeScript typing and is error-prone. CLAUDE.md explicitly wants
  "static content as typed modules in `src/data/`". Locale-keyed typed
  modules give bilingual content *and* type safety (decision D2).

### File Structure

- `Locale` type: reuse the **existing** `export type Locale` from
  `@/i18n/routing` (no new file ‚Äî defect F1). Pages narrow `locale` via
  `hasLocale(routing.locales, locale)` from `next-intl` (doc-confirmed, also
  a type guard ‚Üí narrows to `Locale`).
- `src/data/` ‚Äî bilingual typed modules using **per-record locale maps**
  (`{ ...invariant, i18n: Record<Locale, {...prose}> }[]`) so DE/EN cannot
  drift in length and invariant fields aren't duplicated (F-Q2).
- `src/app/[locale]/<route>/page.tsx` ‚Äî one server component per route; each
  owns its `generateMetadata`. Contact splits page (server, metadata) +
  `ContactForm.tsx` (client) + `actions.ts` (server action).
- `src/app/{robots,sitemap}.ts` ‚Äî site-wide; sitemap uses next-intl
  `getPathname` + `alternates.languages` (doc-confirmed pattern).
- Hover effect: **CSS-only** `.hover-card` rule in `globals.css` on a
  server-rendered `<Link>` (no client component ‚Äî F-Q5). No new
  `src/components/ui/*` file.
